🤖 Claude Code 15项技能速查手册

📱 技能1：移动端编程控制

使用方法

适用场景

  • 通勤时让 CC 在家中的电脑上处理简单任务
  • 长任务跑着，自己出门了，路上追加指令
  • 查看进度，不打断主任务

真实案例

Boris 在沙发上用手机让 CC 处理了一个简单任务，完成后手机收到通知

🔄 技能2：远程会话控制

使用方法

适用场景

  • 公司电脑启动会话做重构 → 下班路上手机审批权限 → 到家后切到个人电脑继续
  • 云端任务做到一半，想在本地终端接手
  • 长任务中途需要审批危险命令，人不在电脑前

真实案例

Boris 说他直接在设置里开了「默认始终开启 Pull」—— 会话不再绑定某台设备，跟着你走

⚡ 技能3：自动化循环（Loop + Perpetual Tasks）

使用方法

Loop vs Perpetual 对比

Loop Perpetual
持久性 退出即消失 重启后依然在跑
时间 单会话内 定时任务
适用 一小时监控部署 每天早上跑测试
过期 1天自动过期 无

真实案例

Boris 跑着一堆自动化循环，”分钟一次是什么概念？你不用再盯着，它会自动回复——这不是工具，是一个真正的同事在值班”

骚操作

🌐 技能4：生命周期钩子（Lifecycle Hooks）

使用方法

适用场景

核心类比

“Git Hooks 是提交后的自动化；Lifecycle Hooks 是代码生成时的自动化”

🖥️ 技能5：桌面应用集成

使用方法

适用场景

  • 在手机上发任务，让 CC 操作桌面上的应用
  • 出门了但需要触发电脑上的某个操作
  • 跨设备工作流

👀 技能6：Browser Extension（最重要技巧）

使用方法

核心价值

作者以前是 CC 的「人类脚手架」：

  • CC 开发完 → 启动测试服务器 → 作者去浏览器看效果 → 截图发回去 → 复制控制台报错粘给它
  • 本质：作者在帮 CC 当它的眼睛

升级后

CC 自己看页面 → 自己截图 → 自己读报错 → 自己迭代修复
作者从「脚手架」变成「验收员」，只需最后看一眼结果

Desktop 端优势

内置浏览器，开箱即用，比第三方方案更稳定可靠

🌳 技能7：会话分支（Session Branching）

使用方法

适用场景

场景 操作
架构设计提了两个方案想都试试 直接 Branch，两个都跑
聊了半小时到分叉路口，想试不同方向 Branch，不丢上下文
以前 只能复制粘贴开新会话，上下文全丢了

作者评价

“这可能是15条里最被低估的功能”

👻 技能8：幽灵上下文（Ephemeral Context）

使用方法

适用场景

场景 以前 现在
长任务中途想问个函数返回类型 等做完/打断任务 直接 /ghost 问
突然想起来要查个东西 开新终端查 直接幽灵问
不确定某个 API 用法 停下手头工作查 边干边问

Boris 原话

Boris 说他同时跑着几十个 CC，靠的就是 Ghost——不污染主上下文，也不打断工作

🏗️ 技能9：Workspaces（沙箱隔离）

使用方法

适用场景

场景 效果
同一仓库同时做不同任务 不会互相踩文件
写坏了也不影响主分支 安全隔离
并行跑多个 CC 干不同事 各走各的，不打架

作者评价

“如果你在同一个仓库里同时跑多个 CC 做不同任务，它们会互相踩文件——Workspace 是一个独立的沙箱”

🚀 技能10：Mission Control（军团式编程）

使用方法

适用场景

场景 操作
大规模代码迁移 自动分析哪些文件可以并行改，同时开几十个各自干
可并行化的工作 Mission Control 自动拆分 + 并行执行

真实案例

举例：把大型项目从 JavaScript 迁移到 TypeScript，以前只能一个个改 → 现在自动分批处理 + 几十个并行跑

作者震惊

“这个功能我还没实际用过，但光想想就觉得恐怖——这不是辅助编程，是军团式编程“

⚡ 技能11：快速启动模式（Skip 自动发现）

使用方法

⚠️ 注意

钥匙串读取也被跳过了，需要通过环境变量或 CLAUDE.md 传入敏感信息

适用场景

  • 写自动化脚本，不需要复杂上下文
  • 大批量调用，每次省几百毫秒，累积下来很可观
  • 裸配置用户也能受益

🔗 技能12：多仓库感知

使用方法

适用场景

场景 效果
前端项目依赖内部库 CC 能同时理解两个仓库的代码
改了库接口 自动去前端项目里更新调用方式
跨仓库代码审查 同时审查多个仓库

Boris 习惯

通常在一个仓库启动会话，然后通过指令把其他仓库加入会话

🎭 技能13：Agent 指令集（Agent Instructions）

使用方法

适用场景

Agent 类型 用途
code-reviewer 专门做代码审查
docs-writer 专门写文档
security-scanner 专门做安全扫描

Boris 原话

“Agent 指令集是一个完整的角色，你甚至可以给它配不同的 MCP Server，让它有不同的能力”

🎤 技能14：语音 + 打字混合输入

使用方法

Boris 的用法

Boris 主要通过说话来写代码
他大部分编码是「说话」而不是打字

适用场景

  • 写长文案/代码，说比打快
  • 突然有灵感，语音捕获比打字快
  • 快速描述需求，不用逐字敲

未来趋势

“编程的交互方式正从『打字』变成『对话』——以前你得准确描述每个技术细节，现在可以像和同事聊天一样说’把那个登录按钮颜色改成蓝色’”

📲 技能15：消息桥接（Message Bridging）

使用方法

适用场景

场景 效果
终端跑着重构 手机追加任务，不用停下手头工作
外部消息源接入 CC iMessage、WhatsApp 等都可以桥接进来
持续运转的工作流 从任何地方发指令

核心意义

“这和 Claude Code 的消息桥接机制异曲同工——让外部消息源接入 AI 产品，是必经之路”

📊 技能速查总表

# 技能 核心命令 一句话总结
1 移动端编程 mcp-notify / CLAUDE.md 手机控制电脑
2 远程会话控制 session attach / pull 会话跟着你走
3 自动化循环 /loop / /perpetual 永不下班的同事
4 生命周期钩子 CLAUDE.md lifecycle 配置 代码生成时的 Git Hooks
5 桌面应用集成 Desktop Controller 手机操控桌面 App
6 Browser Extension 安装扩展 CC 自己当自己的眼睛
7 会话分支 /branch 分叉路口不丢上下文
8 幽灵上下文 /ghost 边干边问不打断
9 Workspaces /workspaces create 沙箱隔离互不干扰
10 Mission Control /mission-control 军团式并行编程
11 快速启动 skip.auto-discovery 启动速度 10x 提升
12 多仓库感知 --repo 参数 跨仓库代码修改
13 Agent 指令集 agents/ 目录 专属角色定制
14 语音+打字混合 空格语音输入 说话写代码
15 消息桥接 mcp-notify 从任何地方发指令

🔥 Boris 核心忠告

**”最重要的技巧是一种验证输出的方式”**（两次强调）

“如果你让工程师做网站但不让他用浏览器，结果会好吗？大概率不行。CC 一样——它需要自己看效果、自己发现问题的反馈循环。”

💡 总结

这15条技巧展现了 Claude Code 正在从「辅助编程工具」进化为「全能编程助手」：

  • 编程不再绑定桌面 - 手机写代码、远程控制、从任何地方发任务
  • 24小时永不下班 - Perpetual Tasks + Loop = 自动运转
  • 并行 = 生产力倍增器 - 同时跑上千个，一个人的产出可相当于工程团队
  • 工具在被重新定义 - 不是辅助编程，是在重新定义编程工具的边界
